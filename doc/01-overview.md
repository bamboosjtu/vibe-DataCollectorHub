## 1. 产品愿景与定位

**Data Collector Hub** 是一个面向**广域数据采集与监控**的插件化数据基础设施，定位于情报分析与态势感知领域的数据底座。

## 2. MVP边界

### MVP核心目标

心中一个数据源（插件）并接入调度，能够定时采集并通过 REST API 提供给下游工具。

### MVP功能边界（摘要）

| 包含 | 不包含 |
|------|--------|
| 1个默认插件 + 定时调度 | 进程级隔离 |
| REST API + RSS/WebSocket + MCP | 多进程/分布式并发 |
| 插件监控告警 | 事件驱动实时流 |
| Web管理界面（只读） | 用户管理和权限控制 |

> 📄 **详细边界定义**：[02-prd.md §4. MVP范围](./02-prd.md#4-mvp范围)

### 成功指标（摘要）

| 类别 | 关键指标 |
|------|----------|
| 可靠性 | 采集成功率 ≥ 99%，调度准时率 ≥ 99.9% |
| 可用性 | 单插件失败不影响其他任务 |
| 性能 | API P99 < 200ms，WebSocket < 1s |
| 扩展性 | 协程级并发 ≥ 2 |

> 📄 **完整SLA定义**：[02-prd.md §5. SLA / 成功指标](./02-prd.md#5-sla--成功指标)

## 6. 关键设计原则

1. **插件独立性**：插件之间不允许有依赖，每个插件必须是独立完整的数据采集工具
2. **逻辑隔离**：配置独立、错误边界独立（非进程级强隔离）
3. **延迟加载**：PluginManager AST解析元数据，用时才import
4. **受控执行**：Semaphore控制并发 + asyncio.wait_for控制超时
5. **单轮询广播**：WebSocket采用单轮询广播模式（详见ADR-003）
6. **三层数据**：raw（原始）→ normalized（规范化）→ feature（特征）
