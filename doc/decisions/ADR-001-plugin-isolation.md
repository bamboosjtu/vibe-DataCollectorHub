# ADR-001: 插件独立无依赖

## 状态

- 状态: 已接受
- 日期: 2026-03-22

## 背景

Data Collector Hub采用插件化架构，需要决定插件之间是否允许依赖关系。

## 决策

**插件之间不允许有依赖关系，每个插件必须是独立完整的数据采集工具。**

```python
class BaseAdapter(ABC):
    dependencies: List[str] = []  # MVP必须为空
```

## 原因

### 1. 故障隔离

- 单个插件失效不影响其他插件执行
- 避免级联故障

### 2. 部署简单

- 无需处理依赖关系图
- 新增插件只需复制一个文件

### 3. 维护清晰

- 每个插件独立版本迭代
- 问题定位更简单

### 4. 性能可控

- 避免依赖加载带来的性能开销
- 延迟加载更容易实现

## 后果

### 正面

- 系统稳定性提升
- 插件开发门槛降低
- 运维复杂度降低

### 负面

- 可能存在代码重复（如共用工具函数）
- 需要通过`_base/`目录提供基础工具类

## 替代方案

考虑过允许插件依赖，但会增加系统复杂度，不符合MVP定位。
